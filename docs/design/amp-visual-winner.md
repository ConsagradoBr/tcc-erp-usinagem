# AMP Visual Winner

## Direção vencedora

Direção escolhida:

**Ceramic Command + Machined Monolith**

Nome de trabalho desta fase:

**Ceramic Monolith**

Esse nome é operacional. Ele existe para orientar design e implementação, não como naming de produto.

## Por que essa direção venceu

Ela combina os dois atributos mais valiosos que apareceram nos estudos:

- a presença industrial, técnica e autoral de `Machined Monolith`
- o refinamento premium, o off-white mineral e o contraste elegante de `Ceramic Command`

Em outras palavras:

- `Machined Monolith` dá a espinha dorsal
- `Ceramic Command` dá o acabamento nobre

## O que herdar de Machined Monolith

- sensação de autoridade
- massa estrutural
- shell mais firme e mais técnico
- leitura de painel de comando
- vínculos visuais fortes entre blocos
- maior presença para números, estados e prioridades
- uso do laranja/cobre como energia funcional

## O que herdar de Ceramic Command

- superfícies internas em off-white mineral
- contraste elegante entre claro e grafite
- silêncio visual
- acabamento mais refinado
- hierarquia mais limpa
- sensação de produto premium bem resolvido

## O que não herdar

### Do Machined Monolith

- azul muito claro e lavado em áreas de apoio
- densidade excessiva
- peso visual a ponto de parecer software duro ou rígido demais

### Do Ceramic Command

- delicadeza excessiva
- leveza que dilui a força industrial
- assepsia visual que reduz a assinatura da AMP

## Paleta conceitual

### Base

- grafite profundo
- chumbo
- cinza aço

### Superfícies

- off-white mineral
- branco cerâmico levemente aquecido
- cinza claro técnico

### Acentos

- cobre queimado
- laranja industrial

### Estados

- verde técnico para concluído/saudável
- âmbar quente para atenção
- vermelho controlado para risco

## Regra de contraste

- o shell pode ser mais escuro e estrutural
- as superfícies de trabalho devem respirar mais
- áreas críticas precisam ter contraste forte
- áreas de apoio devem ser neutras, nunca lavadas
- não usar azul bebê ou tons frios claros que enfraqueçam legibilidade e presença

## Regra de materialidade

- menos vidro cenográfico
- mais superfície sólida bem acabada
- blur só quando reforçar profundidade
- brilho e transparência devem ser discretos
- aparência geral: metal de precisão + cerâmica técnica + software premium

## Regra de composição

- o sistema deve parecer montado em chassis
- o dashboard não pode ser um mosaico de widgets
- o header deve parecer barra de comando
- a sidebar deve parecer coluna estrutural
- painéis internos devem variar em peso, mas pertencer à mesma família

## Regra para dashboard

O dashboard vencedor deve comunicar:

- prioridade operacional
- pendências reais
- impacto financeiro
- vínculo entre cliente, orçamento, OS e financeiro

Ele deve parecer:

- mais painel de operação do que vitrine de KPI
- mais produto autoral do que template de admin

## Regra para telas internas

### Tabelas

- claras, fortes e premium
- com leitura rápida
- com ações contextuais bem resolvidas

### Formulários

- grupos bem definidos
- contraste confortável
- visual técnico, não burocrático

### Modais

- curtos, precisos e elegantes
- sem parecer janela genérica do sistema

## Regra de interação

- animações curtas
- transições firmes
- destaque claro para próxima ação
- sensação de encaixe e precisão, nunca de brincadeira ou adorno

## Critérios para a implementação

Toda primeira implementação da linguagem vencedora precisa mostrar:

- shell coerente com a nova direção
- dashboard com hierarquia forte
- contraste off-white + grafite funcionando bem
- presença industrial sem perder refinamento
- ausência total de cores frias lavadas que apaguem o conteúdo

## Próximo passo recomendado

Aplicar `Ceramic Monolith` primeiro em:

1. shell global
2. dashboard
3. uma tela de lista
4. um modal ou formulário-chave

Assim a linguagem é validada cedo antes de se espalhar pelo sistema inteiro.
